# Security policy

## Supported version

Security fixes are applied to the current `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a vulnerability. Use GitHub’s private vulnerability reporting feature on this repository and include:

- affected files and behavior;
- reproduction steps or a proof of concept;
- potential impact;
- any suggested mitigation.

This project is a static client-side visualization. It does not require credentials, collect personal data, or operate a backend service.
